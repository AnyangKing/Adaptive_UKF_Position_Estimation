# Data and code availability plan

The manuscript is simulation-based, so reviewers may ask for the scripts, configurations, and result files that support the headline numbers.

## Recommended policy

Preferred:

> Public repository plus archived release DOI.

Why:

- Strongest reproducibility posture.
- Fits the Sensors emphasis on reproducibility and data/code availability.
- Makes the simulation-only nature of the work more defensible.

Fallback:

> Supplementary ZIP plus data/code available on request.

Use only if repository cleanup takes too long.

## Minimum content to expose

- canonical channel/simulation configuration;
- carrier schedule definition;
- static 600 m validation results;
- moving-target whitening results;
- quasi-static boundary results;
- figure-generation scripts;
- method parameter audit from folder 93.

## Human decisions required

- whether the GitHub repository can be made citable/public as the data/code source;
- whether to create a DOI archive using Zenodo/Figshare/OSF or similar;
- whether any files must be excluded before public release.
