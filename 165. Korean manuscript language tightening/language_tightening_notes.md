# Language tightening notes

## 원칙

이번 문장 다듬기는 내용을 키우지 않는다.
좋아 보이는 claim을 새로 추가하지 않고, 이미 검증된 내용만 더 읽기 쉽게 정리한다.

## 주요 수정 방향

- "출발했다", "보여주었다", "드러났다"가 반복되는 설명문 톤을 줄였다.
- 초록에서 문제, 원인, 방법, 결과, 경계가 빠르게 보이도록 순서를 정리했다.
- 서론에서 필터 개선 실패와 carrier sensitivity 발견 사이의 전환을 더 선명하게 만들었다.
- 표와 그림 callout은 유지했다.

## 변경하지 않은 claim

- 본문 중심 claim은 static 600 m carrier-agile whitening이다.
- moving target은 성능 개선 claim이 아니다.
- quasi-static은 0.005 m/s까지만 안전하다.
- TOA guard는 post-hoc pilot이므로 독립 검증 전까지 성능 claim이 아니다.

## 영어 원고 반영 시 주의

영어 원고로 옮길 때는 이번 한글 문장을 직역하지 말고, 같은 논리 순서를 유지하면서 IEEE 논문 문체로 바꾼다.
특히 "모든 이동 표적 개선"처럼 읽힐 수 있는 문장은 반드시 피한다.
