# 공부 노트: content-addressed supplement manifest

보충자료 준비에서 중요한 것은 파일을 한곳에 복사하는 행위보다, **어떤 원본의 어떤 버전이 논문 수치를
지지하는지**를 고정하는 것이다. 경로만 적으면 원본이 바뀌어도 알아채기 어렵지만, byte size와 SHA256을
함께 기록하면 한 비트의 변경도 검출할 수 있다.

이번 구조는 allowlist 방식이다. 포함할 데이터·코드·그림을 명시하고, 원고·로그·캐시·운영 문서는
denylist로 차단한다. 따라서 `git add .`나 전체 폴더 복사로 인한 비의도적 공개 위험을 줄인다.

dry run과 실제 archive를 분리한 이유는 공개 범위가 아직 사람의 결정 사항이기 때문이다. 현재 단계에서는
manifest만 고정하고, 공개 정책과 저널 규정이 정해진 뒤 동일 manifest를 사용해 ZIP을 조립하는 것이 안전하다.
