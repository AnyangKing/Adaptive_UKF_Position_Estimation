## 목적

v1 품질체크 #4·v2 잔여 ②의 수행: manuscript v2가 주장하는 Method/프로토콜 수치를 **실제 채택
코드와 기계적으로 대조**해, (a) 수치·서술 불일치를 찾고 (b) 재현성에 필요한 누락 파라미터를
정리해 v3용 패치로 만든다. 근거는 헤드라인 결과를 만든 폴더들(61 정지 검증, 63 이동, 82 준정지)과
공통 파이프라인 파일의 커밋된 원문이다.

## 방법

- `extract_method_facts.py` — 61/63/82의 config·runner·wrapper 파일을 regex로 파싱(모듈 import
  충돌 회피, "커밋된 파일 그대로"가 근거)하고 82는 실행 당시 config가 기록된 결과 JSON을
  authoritative로 사용. `results/method_facts.json` 산출.
- 추출값을 v2 전문과 대조해 `method_facts_table.md`(대조표)와
  `manuscript_v2_section2_patch.md`(P1~P4 패치 전문) 작성.
- 검증: `python test_diagnostic.py` (핵심 상수·프로토콜 12항목).
