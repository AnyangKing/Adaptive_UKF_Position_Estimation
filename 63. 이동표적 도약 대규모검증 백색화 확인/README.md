# 63. 이동표적 도약 대규모검증 백색화 확인

## 목적

62번에서 주파수 도약의 단순 설명, 즉 “관측 창의 총 위상 스윙이 작을수록 이득이 크다”는 가설은 기각됐다. 그러나 fixed 32 kHz에서는 ping 간 DOA 편향이 시간상관을 가진 상태 오차처럼 누적되고, 30~34 kHz 도약은 이 편향을 ping마다 다른 부호/크기의 백색성 오차로 바꿔 UKF가 평균낼 수 있게 만든다는 새로운 해석이 나왔다.

이번 63번은 이 해석을 사전등록 방식으로 직접 검증한다.

- M1: 이동 표적 정착 추적에서도 주파수 도약의 pooled RMSE 이득이 0보다 큰가?
- M2: 고도각 오차 시퀀스의 lag-1 자기상관이 fixed보다 hop에서 유의하게 작아지는가?
- M3: 조건별 이득 방향은 62번의 탐색 결과와 일관적인가?

## 방법

- 거리: 600 m
- 조건: `radial_0.05`, `radial_1.0`, `tangential_1.0`, `tangential_1.0+vz0.08`
- 조건당 독립 기하: 16개, 총 64개 이동 궤적
- ping: 20개, 정착 RMSE는 후반 10개 ping
- 비교:
  - fixed: 32 kHz
  - hop: 30~34 kHz 균등 20개 carrier, 61/62번 정책 동결
- 추정기: 기존 conditional adaptive-R UKF 그대로 사용
- 백색화 지표: GT는 평가 label로만 사용하여 측정 고도각 오차 `z[9] - h(x_true)[9]`의 lag-1 자기상관 계산
- 실행: `python run_moving_validation.py`
- 검증: `python test_diagnostic.py`

장시간 실행 중 끊겨도 이어서 돌릴 수 있도록 `results/moving_validation_partial.json`에 run 단위 중간 저장을 추가했다.

## 결과

### M1. 이동 표적 pooled RMSE 이득

| 항목 | 값 |
|---|---:|
| fixed 대비 hop 평균 이득 | -0.103 m |
| median 이득 | +0.406 m |
| 95% bootstrap CI | [-2.814, +2.801] m |
| Wilcoxon p(gain > 0) | 0.301 |
| 개선 비율 | 57.8% |
| n | 64 |

이동 표적 전체에 대해 “주파수 도약이 RMSE를 유의하게 개선한다”는 M1은 기각한다. median과 개선 비율은 약간 양수지만, 평균과 신뢰구간은 이득을 확정하지 못한다.

### M2. 백색화 메커니즘

| 항목 | fixed | hop |
|---|---:|---:|
| mean lag-1 autocorr | +0.470 | -0.208 |
| median lag-1 autocorr | +0.489 | -0.351 |

- Wilcoxon p(fixed lag-1 > hop lag-1) = `5.56e-10`
- lag-1 감소 비율 = 84.4%
- lag-1 감소량과 RMSE 이득의 Spearman 상관 = ρ=0.298, p=0.0166

백색화 메커니즘은 매우 강하게 통과했다. 주파수 도약은 실제로 고도각 오차의 ping 간 자기상관을 크게 낮춘다. 또한 자기상관 감소가 클수록 RMSE 이득도 커지는 약한 양의 상관이 있다.

### M3. 조건별 결과

| 조건 | fixed RMSE | hop RMSE | 평균 이득 | median 이득 | 개선 비율 | p |
|---|---:|---:|---:|---:|---:|---:|
| radial 0.05 | 13.29 | 11.67 | +1.62 | +1.13 | 0.75 | 0.096 |
| radial 1.0 | 17.61 | 12.55 | +5.06 | +0.36 | 0.56 | 0.126 |
| tangential 1.0 | 10.00 | 11.71 | -1.71 | +0.74 | 0.69 | 0.281 |
| tangential 1.0 + vz | 12.82 | 18.20 | -5.38 | -2.62 | 0.31 | 0.967 |

radial 조건은 평균적으로 이득 방향이지만 유의하지 않다. tangential+vz 조건에서는 도약이 명확히 악화된다. 즉 백색화는 관측오차의 시간상관을 줄이지만, 이동 동역학/수직기하/도약 관측의 결합이 항상 위치 RMSE를 낮추지는 않는다.

## 판정

**진단 통과, 성능 채택 보류.**

이번 결과는 “주파수 도약이 이동 표적에서도 무조건 성능을 올린다”는 주장을 지지하지 않는다. 따라서 63번 결과만으로 frequency-agile pinging을 이동 표적 주 알고리즘으로 채택하면 안 된다.

대신 더 중요한 사실이 확인됐다.

> frequency-agile pinging은 멀티패스 유도 DOA 편향의 ping 간 자기상관을 강하게 낮춘다. 그러나 이 백색화 효과가 위치 RMSE 개선으로 이어지려면 동역학 조건과 필터의 관측 가중/게이팅이 함께 맞아야 한다.

따라서 논문 방향은 다음처럼 좁혀야 한다.

1. 정지/준정지 또는 radial 성분이 지배적인 장거리 추적에서는 도약이 유망하다.
2. vertical 성분이 섞인 이동 추적에서는 도약이 오히려 outlier 악화를 만들 수 있다.
3. 다음 단계는 단순 fixed vs hop이 아니라, 백색화는 유지하면서 위험 조건에서는 R을 조절하거나 hop을 제한하는 **whitening-aware adaptive-R / schedule gating**이다.

## 다음 연구 제안

64번은 “도약 사용 여부를 항상 켜는 방식”이 아니라, hop으로 인해 백색화가 발생하되 innovation이 커지는 조건을 감지해 DOA R을 늘리는 방법을 검증한다.

가능한 가설:

- H1: hop 관측의 lag-1 자기상관 감소는 유지하되, NIS/innovation 기반 adaptive-R을 결합하면 tangential+vz 악화를 줄일 수 있다.
- H2: radial 계열에서는 hop 유지, vertical 성분이 의심되는 경우 DOA R inflation 또는 fixed fallback을 적용하면 pooled 이동 RMSE가 양수로 전환된다.

주의: 63번은 결과적으로 “이동 표적 전체 성능 개선”은 실패했지만, 메커니즘 검증으로는 강한 결과다. 실패를 숨기지 말고 “백색화만으로 충분하지 않다”는 전환점으로 사용한다.
