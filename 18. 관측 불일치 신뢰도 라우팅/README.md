# 18. 관측 불일치 신뢰도 라우팅

## 문제

기존 adaptive R은 GCC와 SRP가 다르면 DOA R을 확대했다. 17번 200 m 진단에서는 SRP
실제 오차가 1.5~2.3°로 안정적인데 GCC-SRP 불일치는 21~29°였다. 이때 기존 규칙은
좋은 SRP를 버리고 같은 GCC에서 만든 나쁜 TDOA를 신뢰하는 반대 행동을 했다.

## 변경

- old: disagreement scale을 DOA block에 적용
- routed: 동일 scale을 TDOA 7×7 block에 적용하고 SRP DOA는 유지
- scale: `min(100, 1 + (disagreement_deg/2)²)`
- TOA와 NIS block soft gating은 동일

SRP가 항상 옳다는 가정이 아니라, 3번 Monte Carlo에서 SRP의 최대오차가 GCC보다 작았고
17번 발산 진단에서도 SRP가 안정적이었다는 독립 근거에 따른 라우팅 가설이다.

신규 seed 180000 계열의 100/200/400/600 m 궤적에서 비교하며 결과로 scale 식을 다시
맞추지 않는다.

## 실행

```powershell
python test_routing.py
python run_compare.py
```

## 최초 결과 (2026-07-03)

| 거리 | 기존 DOA 감쇠 RMSE | TDOA 라우팅 RMSE | 변화 | 평균 불일치 |
|---:|---:|---:|---:|---:|
| 100 m | **0.893 m** | 0.894 m | -0.14% | 0.46° |
| 200 m | 6.245 m | **6.146 m** | +1.59% | 3.33° |
| 400 m | **21.696 m** | 25.362 m | -16.90% | 2.59° |
| 600 m | **21.071 m** | 21.132 m | -0.29% | 0.45° |

항상 TDOA로 불일치 scale을 라우팅하는 정책은 채택하지 않는다. 17번 발산 사례의
21~29° 불일치는 극단적이었고, 신규 test의 정상 0.45~3.33° 불일치에서는 TDOA 정보
약화가 오히려 손실이었다.

결론은 estimator 불일치를 연속적으로 한쪽 관측에 무조건 배정하면 안 된다는 것이다.
다음 후보는 validation에서 정한 큰 불일치 문턱을 넘을 때만 TDOA를 감쇠하고, 평상시에는
기존 정책 또는 full 관측을 유지하는 조건부 라우팅이다. 18번 test는 이후 개발 데이터로
간주하며 새 정책은 새로운 seed에서 평가한다.
