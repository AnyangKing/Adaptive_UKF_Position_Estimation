# 115. Reproducibility package audit

## 목적

교수님 복귀 전, 논문의 핵심 주장과 그림이 실제 코드·결과 파일로 재현 가능한지 다시 묶어 확인했다.
112번 `보충자료 매니페스트`가 “무엇을 넣을지”의 목록이었다면, 이 폴더는 한 단계 더 좁혀서
**원고 claim → 근거 JSON/CSV → 실행 스크립트 → 재현 상태 → 남은 위험**을 투고 방어용으로 정리한다.

새 실험은 하지 않았다. 이미 동결·검증된 58·61·63·82·45·93·95·101·108~112번 산출물을 감사했다.

## 결론

**재현성 패키지의 과학적 뼈대는 준비되어 있다.** 핵심 수치들은 로컬 결과 JSON과 일치하고, 각 검증
폴더가 실행 스크립트와 공통 파이프라인 파일을 자체 포함한다.

다만 투고 직전에는 아래 세 가지를 별도로 닫아야 한다.

1. 결과 JSON/CSV는 `.gitignore` 때문에 GitHub에 없으므로, 보충자료 ZIP이나 DOI archive로 따로 묶어야 한다.
2. 현재 원고는 로컬 `paper/` 폴더에 있으며 GitHub에는 올리지 않는다. 논문 파일은 Git 추적 대상이 아니다.
3. 110번의 `latexmk` 단일 명령 설명은 과거 루트 LaTeX 상태가 섞여 있다. 현재 PC에서는 Perl 부재로
   `latexmk`가 막힐 수 있으므로, 검증된 빌드 경로는 `paper/`에서 `pdflatex → bibtex → pdflatex ×2` 수동 체인이다.

## 핵심 산출물

- `claim_to_artifact_matrix.md`: 논문 주장·그림별 코드/데이터 근거표.
- `rerun_commands.md`: 다른 AI나 연구자가 재실행할 때의 명령·주의사항.
- `supplement_gap_closure.md`: 보충자료 패키지를 만들 때 남은 결정과 금지 항목.

## 판정

115번 기준으로, 교수님 부재 중 다음으로 할 수 있는 가장 가치 있는 작업은 116번
`Manuscript claim boundary audit`다. 이유는 재현성 근거는 이미 충분히 모였고, 이제 원고 문장이
그 근거보다 앞서나가지 않도록 과장·위험 표현을 잡는 단계가 필요하기 때문이다.
