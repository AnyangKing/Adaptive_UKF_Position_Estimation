# 2. 신호 기반 수중 채널

## 목적

관측값에 독립 가우시안 오차를 직접 더하지 않고, 수중 채널을 통과한 8센서 원시
신호로부터 TOA와 배열 DOA를 추정한다. `1. 기본 UKF`는 변경하지 않았다.

## 모델

- 송신 신호: 중심 32 kHz, 대역폭 12 kHz, 길이 10 ms LFM chirp
- 표본화율: 192 kHz
- 환경: 수심 100 m, 배열 중심 수심 30 m
- 채널: 직접파 + 해수면 1회 반사 + 해저 1회 반사
- 반사계수: 해수면 -0.90, 해저 +0.60
- 전달손실: 구면 확산과 Thorp 주파수 흡수 근사
- 시간변화: 방사속도에 의한 1차 Doppler time scaling
- 주변잡음: 저주파 선박/파랑과 고주파 열잡음의 경향을 합성한 유색잡음
- 기본 SNR: 20 dB

경로는 평면 경계의 image-source 방법으로 계산한다. 이 모델은 Bellhop의 대체물이
아니라 신호 처리와 필터 연결을 검증하기 위한 첫 물리 기준선이다.

## 좌표계 변경

기존 8센서 형상은 그대로 유지하되, 배열의 기하학적 중심을 `[0, 0, -30] m`에 둔다.
해수면은 `z=0`, 해저는 `z=-100 m`이다. 이는 실제 수심과 반사를 정의하기 위해
필요한 큰 변경이므로 새 시행착오 폴더로 분리했다.

## 관측 추정

- TOA: 알려진 chirp와 센서 신호의 matched-filter 상관에서 첫 유효 피크 검출
- DOA: 8개 TOA의 센서간 차이를 평면파 최소제곱으로 한 번에 결합
- 출력 DOA: 배열 전체의 방위각 1개와 고도각 1개

단일 센서별 DOA 8쌍을 생성하던 이전 방식은 실제 배열 관측으로 사용하지 않는다.

## 실행

```powershell
python test_channel.py
python run_smoke.py
```

## 한계와 다음 단계

- 평면 해수면/해저와 일정 음속을 가정한다.
- 해저 반사계수는 주파수·입사각·퇴적층에 따라 변하지만 현재는 상수이다.
- 잡음 PSD는 실제 해역 계측자료로 보정되지 않은 상대 형상이다.
- 산란, 표면 거칠기, 음속 프로파일 굴절, 센서 응답 오차는 아직 없다.
- 실제 해역 파라미터가 정해지면 Bellhop 또는 검증된 채널 출력과 교차검증한다.

다음 폴더에서는 이 추정 관측 `[TOA/TDOA, array azimuth, array elevation]`을 UKF에
연결하고 matched-filter peak quality 및 잔차로 측정 공분산을 구성한다.

## 최초 smoke test 결과 (2026-07-03)

소스 `[100, 20, -20] m`, SNR 20 dB, 방사속도 0.5 m/s, seed `20260703` 조건이다.

- TOA RMSE: 12.266 us
- 기준 센서 대비 TDOA RMSE: 10.841 us
- 방위각 오차: 0.020 deg
- 고도각 오차: 16.611 deg
- 단위/smoke test: 5개 통과

고도각 오차가 큰 것은 실패를 숨긴 결과가 아니다. 이 시나리오에서는 직접파와 약
7 ms 늦은 해수면 반사파가 10 ms chirp 구간에서 겹치며, 센서의 수직 aperture도
0.079 m에 불과하다. 따라서 개별 matched-filter peak의 작은 편향이 평면파 고도각
해에 크게 증폭된다. 다음 시행착오에서는 GCC-PHAT, 다중 peak 분리, 광대역 MUSIC
중 어느 방법이 이 조건에서 타당한지 비교해야 한다.
